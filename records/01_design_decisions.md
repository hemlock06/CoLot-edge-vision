# 01 — 설계 결정

## D1. 범위: 핵심 AI 3개, 인프라 스프롤 배제
코랏 카스토퍼가 실제로 한 3가지(번호판 인식·불법주차 경고·휘도 환경적응)를
하나의 엣지 파이프라인으로 구현. MQTT/InfluxDB/Grafana/CI 같은 운영 인프라는
**의도적으로 제외** — 본인이 이해·방어 가능한 규모를 유지하기 위함(포트폴리오 우선순위).

## D2. ① 경량화 — 정적 INT8 (동적 양자화 폐기)
- 1차 시도: `quantize_dynamic`(가중치 INT8). 결과 **지연 198ms로 역행** —
  동적 양자화는 matmul 위주(transformer)에 유효, YOLO의 conv는 가속되지 않고
  quantize/dequantize 오버헤드만 추가.
- 채택: **정적 양자화**(`quantize_static`, QDQ, per-channel) + 50장 캘리브레이션.
  opset 13(per-channel DequantizeLinear `axis` 지원) 필요.
- 결과: 3.4MB(7.2×↓), 검출 0.96 유지. 단 INT8 CPU 지연 이득은 VNNI/NPU 의존(→ 04).
- 엣지 추론은 ultralytics 대신 **onnxruntime + YOLOv8 후처리 직접 구현**(`OnnxYolo`)
  으로 FP32/INT8을 동일 경로에서 공정 비교.

## D3. ② 이상탐지 — 룰 우선 + IsolationForest 보강
- 예약/결제 원장 대조는 **결정론적**(무단=미예약, 초과=결제초과+grace) → 룰이 P=1.0.
- ML의 가치는 *원장이 정상인데 센서만 약하게 이상한* subtle 고장(약flicker·짧은ghost):
  정상 세션으로 적합한 IsolationForest가 통계 이상으로 포착 → 고장 재현율 70.5→88.6.
- **트레이드오프 인정**: ML이 정상 일부를 오탐해 정밀도 1.0→0.87. 운영상 룰=즉시 제재,
  ML=2차 검토 큐로 분리하는 것이 합리적. F1만 보면 룰 우선이 높음 — 과설계 경계.
- contamination 0.045→0.03으로 정밀도 보정(0.83→0.87) 후 채택.

## D4. ③ 환경적응 — 휘도 + 악천후, 학습 분류기
- 국방 표적감시 이전기술(휘도 기반 환경식별)을 재현 → RandomForest 8클래스.
- 룰 라벨러는 **베이스라인/폴백**으로만(0.47); 학습 분류기가 0.972로 우위 → "학습의 가치".
- **누수 차단**: train/test를 *장면 단위*로 분리(같은 장면의 환경변형이 양쪽에 섞이지 않게).
- 악천후(비·안개·눈)는 **원 휘도 특허 범위 밖의 실외 로버스트니스 확장**으로 명시.
  안개는 dark-channel prior(dehazing 원리)·채도로 식별 — 원리적 피처라 룰로도 일부 포착.
- 에너지 모델: 모션 게이팅(차량 부재 시 절전) + 휘도별 모드 + 판독불가 시 OCR 생략.

## D6. 보강 4종 — '주장-구현 갭' 우선 (신규 기능보다 약속 이행)
포폴이 *말로 주장하던* 부분을 실제로 구현해 방어력을 높임(과설계 대신 갭 해소).
1. **#3 OCR 교정**: EasyOCR allowlist(번호판 charset) + 숫자 혼동맵 + 포맷 코어션.
   한글(나/너)↔숫자 혼동은 단독 교정 불가 → #1 투표로 회수(상보).
2. **#1 다중프레임 투표**: ③ 정책이 '악천후 다중판독'을 명시했으나 단일 프레임만
   구현돼 있던 갭. 문자 단위 다수결(유효 포맷 우선)로 비·눈 OCR +13~17%p.
3. **#2 실시간 조기 경보**: ②는 출차 후 세션 판정 → '불법주차 경고'의 실제 의미
   (주차 중 경보)와 불일치. 분 단위 스트림에서 위반 즉시 경보, 선행시간 측정.
   presence/원장 기반 위반(무단·초과·stuck)만 조기 탐지; transient(ghost·flicker)는
   배치가 담당(상보적 — 정직히 명시).
4. **#4 분단위 정산**: AI(번호판·점유)를 코랏의 PM 차별점(*분단위 정산 vs 30분 블록*)에
   연결. 무단=과태료·초과=할증. 연구가 아닌 제품/사업 로직임을 표기.

## D5. 정직성 5원칙
1. 비공개 데이터를 지어내지 않음 — 합성으로 구조만 재현, README/decisions에 명시.
2. 하드웨어 의존성(INT8↔VNNI)을 측정으로 드러냄.
3. 정밀도-재현율 트레이드오프를 숨기지 않음.
4. 합성 분리도가 높아도(예: 환경분류) 누수 차단·노이즈로 난이도를 정직하게 부여.
5. 역할 경계 — "기성 모델 적용·시스템 통합"이지 "모델 연구"가 아님을 표기.
