# 04 — 이슈 / 한계 / 향후

## 한계 (정직)
- **INT8 CPU 지연**: 이 데스크톱 CPU(VNNI/AVX512-VNNI 미지원)에서 정적 INT8(36.5ms)이
  FP32-ONNX(14.1ms)보다 느림. INT8의 지연 이득은 VNNI CPU·NPU·모바일 가속기에서 발현.
  → 현 하드웨어 기준: **지연 최적 = ONNX-FP32, 크기 최적 = INT8**.
- **이상탐지 정밀도-재현율 트레이드오프**: IForest가 센서고장 재현율을 올리나(70.5→88.6)
  정상 일부 오탐으로 정밀도 1.0→0.87, 전체 F1은 룰만(0.914)보다 낮음(0.880).
  ML은 2차 검토 신호로 두는 것이 운영상 타당.
- **합성↔실데이터 갭**: 배경이 균일한 합성 노면, 기성 EasyOCR 사용. 실제 분포 이동·
  악천후 실사·다양한 번호판 서체는 미반영. 수치는 *구조 검증*이지 실배포 성능 보장 아님.
- **OCR 오인식 패턴**: 일부 한글(나/너)을 숫자 4로 오인 — 포맷 정규식으로 일부 필터.

## 향후 (스케일업 시)
- 정적 INT8 → TFLite/NNAPI·TensorRT(엣지 NPU) 변환으로 지연 이득 실현.
- 실 번호판 공개셋(AIHub 등) 또는 도메인 적응으로 합성 갭 축소.
- ② 스트리밍화: 출차 이벤트 단위가 아닌 분 단위 점유 스트림에서 조기 경보(현재는 세션 종료 시 판정).
- 악천후 실사 증강(렌즈 물방울 물리 모델·야간 우천 결합).
